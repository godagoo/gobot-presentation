Your Always-On AI Assistant

Claude-powered Telegram bot with persistent memory, multi-agent reasoning, and 24/7 availability

What is GoBot?

An always-on personal AI assistant that runs on Telegram, powered by Claude with persistent memory, multi-agent routing, proactive check-ins, voice capabilities, and morning briefings.

Three deployment modes: local desktop (free with Claude subscription), VPS (pay-per-token), or hybrid (combines both).

Hybrid Architecture

Telegram → VPS (always on)
↓
Is local machine awake?
├─ YES → forward to local (free, Claude Code)
└─ NO → process on VPS (paid, Anthropic API)
↓
Supabase (shared database)

Key principle: Same code everywhere. One codebase for local, VPS, and hybrid modes.

03 // SETUP

Getting Started

Setup Guided by Claude Code

No programming experience needed. Claude Code walks you through everything:

git clone https://github.com/autonomee/gobot.git
cd gobot
bun install
claude    # Opens Claude Code for guided setup

Claude Code then guides:

  • Creating Telegram bot via BotFather
  • Setting up Supabase project
  • Personalizing profile and agents
  • Testing the bot
  • Configuring always-on services
  • Deploying to VPS (optional)

Prerequisites

  • Bun runtime — Modern TypeScript runtime
  • Claude Code CLI — For local mode
  • Telegram account — Free messaging platform
  • Supabase account — Free database tier
  • PM2 — For Windows/Linux always-on (macOS uses launchd)

All free and open-source except Claude subscription ($20/month)

Design Principles

  • Single user, full power — Personal AI, not SaaS
  • Graceful degradation — Missing APIs disable features, don't crash
  • Process isolation — Each service independent
  • Context over memory — Claude gets full context each call
  • Local fallback — Works offline with local LLMs
  • Same code everywhere — One codebase for all modes
